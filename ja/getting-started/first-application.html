はじめてのアプリケーションを作成¶

この最初のガイドでは、 NEM においての一般的な概念 を紹介します。目新しいことを見つけたら、提供されたリンクをクリックして詳細を読むことをおすすめします。

背景¶

モザイク は物体、チケット、クーポン、株式に相当するもの、あなたの暗号通貨さえも、いかなる資産をブロックチェーン上に表現するために使用されます。

このガイドを終了したら、あなたはモザイクを作成して、はじめての 転送トランザクション を送ることを15分以下で実施できるようになります。

一見すると、短期間でこれだけのことができるとは想像しにくいかもしれません。ですが、NEM のツール郡を使用するとブロックチェーンの開発の複雑さを削減でき、開発者に製品の開発へ集中させることでそれが可能になります。

前提条件¶

  • 入門セクション を終了していること
  • テキストエディタ または IDE
  • NEM2-SDK と CLI
  • XEM を保有したアカウント

それではコードを見ていきましょう¶

残高を確認してください

テストネットXEMを保有したアカウントの読み込みは済ませましたか？まだ終わっていないのであれば 入門セクション に戻ってください。

あなたのアカウントに XEM が入ってきているかを確認しましょう。

$> nem2-cli account info

このコマンドの実行後に、同じような行がスクリーンに表示されるでしょう:

New Account: SCVG35-ZSPMYP-L2POZQ-JGSVEG-RYOJ3V-BNIU3U-N2E6

[…]

モザイク

3628d0b327fb1dd8: 1000000

アカウントは 1,000,000 XEM を保有しています。もし mosaics の次の行が空であるならば、前のチュートリアル に従ってください。

ブロックチェーンの監視

アカウントはトランザクションを通じてブロックチェーンの状態を変更します。アカウントが正しく構成されているトランザクションをアナウンスすると、サーバーは OK のレスポンスを返します。

OK のレスポンスを受信することは、トランザクションが正当であることを意味するわけではなく、まだブロックには含まれていないことを意味します。トランザクションをアナウンスする前に監視を始めておくのがよいでしょう。

ターミナルを3つ開くことをおすすめします。バリデーションエラーのある アナウンスされたトランザクションの監視 。

$> nem2-cli monitor status

未承認 の監視はトランザクションがネットワークに到着したことを確認できますが、まだブロックには含まれてません。

$> nem2-cli monitor unconfirmed

トランザクションが含まれると confirmed の下に表示されるでしょう:。

$> nem2-cli monitor confirmed

ネームスペースの作成

モザイクを作成する前に、ネームスペースを取得する必要があります。

ネームスペース はチェーン上の資産のための一意なドメインです。インターネット上のドメインのファイルの関係に類似していることに例えるのが、これを理解するのに最も簡単です。ドメインのアドレスがルート(最下層)において一意であることを考えてみてください。

モザイクはドメイン上にホストされているファイルのようなもので、資産として表現されます。ウェブサイトやディレクトリと同様に、モザイクは他のドメインの他のファイルと同じ名前を持つことができます。それでも、ネームスペース + モザイクは常に一意になります。

まず、あなたの好きな名前でネームスペースを登録しましょう。よくある方法として、あなたの会社や自分の名前を使用します。このケースでは、foo というネームスペースを登録します。この名前が有効であるか確認してみましょう。

$> nem2-cli namespace info --name foo

そのネームスペースは有効ですか？ネームスペース名とブロック数で数えたレンタル期間を指定して取得してください。

注釈

Catapultにおいて、NEM ブロック生成は平均 15 毎秒で完了します。

$> nem2-cli transaction namespace --name foo --rootnamespace --duration 1000

アカウントのトランザクションを監視しているターミナルで何が起きたかを確認しましたか？トランザクションはまず unconfirmed の下に表示され、その後 confirmed になります。

モザイクの作成

モザイクの名前を選んでください。モザイク名は最大 64 文字で一意なドメイン名でなければなりません。

モザイクは token と呼び、その親ネームスペースは foo であり、供給量は 1,000,000 となります。

のちのトランザクションでは、このモザイクを foo:token として参照してください。

$> nem2-cli transaction mosaic --mosaicname token --namespacename foo --amount 1000000 --transferable --supplymutable --divisibility 0 --duration 1000

はじめての転送トランザクションの送信

おめでとう！これであなたはこのモザイクの所有者です。 転送トランザクション をアナウンスすることで、幾分かの foo:token を他のアカウントへ送ることができます。これは NEM の中で最も一般的なアクションです。

転送トランザクションは３つの不可欠な属性によって成り立ちます:

  • 受信アドレス
  • メッセージ、暗号化することも可能です
  • モザイク配列

この例では、 SC7A4H-7CYCSH-4CP4XI-ZS4G2G-CDZ7JP-PR5FRG-2VBU へ 10 foo:token を送りました。また、 my first transfer transaction! のようにメッセージを添付することもできます。新しくアカウントを作成して、受信アドレスを変更してみましょう。

import {
    Account, Address, Deadline, UInt64, NetworkType, PlainMessage, TransferTransaction, Mosaic, MosaicId,
    TransactionHttp
} from 'nem2-sdk';

const transferTransaction = TransferTransaction.create(
    Deadline.create(),
    Address.createFromRawAddress('SC7A4H-7CYCSH-4CP4XI-ZS4G2G-CDZ7JP-PR5FRG-2VBU'),
    [new Mosaic(new MosaicId('foo:token'), UInt64.fromUint(10))],
    PlainMessage.create('my first transfer transaction!'),
    NetworkType.MIJIN_TEST
);

import io.nem.sdk.model.account.Address;
import io.nem.sdk.model.blockchain.NetworkType;
import io.nem.sdk.model.mosaic.Mosaic;
import io.nem.sdk.model.mosaic.MosaicId;
import io.nem.sdk.model.transaction.Deadline;
import io.nem.sdk.model.transaction.PlainMessage;
import io.nem.sdk.model.transaction.TransferTransaction;

import java.math.BigInteger;
import java.util.Arrays;

import static java.time.temporal.ChronoUnit.HOURS;

final TransferTransaction transferTransaction = TransferTransaction.create(
    Deadline.create(2, HOURS),
    Address.createFromRawAddress("SC7A4H-7CYCSH-4CP4XI-ZS4G2G-CDZ7JP-PR5FRG-2VBU"),
    Arrays.asList(new Mosaic(new MosaicId("foo:token"), BigInteger.valueOf(10))),
    PlainMessage.create("my first transfer transaction!"),
    NetworkType.MIJIN_TEST
);

const nem2Sdk = require("nem2-sdk");

const Address = nem2Sdk.Address,
    Deadline = nem2Sdk.Deadline,
    Account = nem2Sdk.Account,
    UInt64 = nem2Sdk.UInt64,
    NetworkType = nem2Sdk.NetworkType,
    PlainMessage = nem2Sdk.PlainMessage,
    TransferTransaction = nem2Sdk.TransferTransaction,
    Mosaic = nem2Sdk.Mosaic,
    MosaicId = nem2Sdk.MosaicId,
    TransactionHttp = nem2Sdk.TransactionHttp;

const transferTransaction = TransferTransaction.create(
    Deadline.create(),
    Address.createFromRawAddress('SC7A4H-7CYCSH-4CP4XI-ZS4G2G-CDZ7JP-PR5FRG-2VBU'),
    [new Mosaic(new MosaicId('foo:token'), UInt64.fromUint(10))],
    PlainMessage.create('my first transfer transaction!'),
    NetworkType.MIJIN_TEST
);

トランザクションは作られましたが、まだネットワークへアナウンスされていません。ネットワークがトランザクションの信憑性を検証できるように、あなたのアカウントでトランザクションに署名します。

const privateKey = process.env.PRIVATE_KEY;

const account = Account.createFromPrivateKey(privateKey, NetworkType.MIJIN_TEST);

const signedTransaction = account.sign(transferTransaction);

final String privateKey = "";

final Account account = Account.createFromPrivateKey(privateKey,NetworkType.MIJIN_TEST);

final SignedTransaction signedTransaction = account.sign(transferTransaction);

const privateKey = process.env.PRIVATE_KEY;

const account = Account.createFromPrivateKey(privateKey, NetworkType.MIJIN_TEST);

const signedTransaction = account.sign(transferTransaction);

署名が終わったら、ネットワークへアナウンスします。

const transactionHttp = new TransactionHttp('http://localhost:3000');

transactionHttp.announce(signedTransaction).subscribe(
    x => console.log(x),
    err => console.log(err)
);

final TransactionHttp transactionHttp = new TransactionHttp("http://localhost:3000");

transactionHttp.announceTransaction(signedTransaction).toFuture().get();

const transactionHttp = new TransactionHttp('http://localhost:3000');

transactionHttp.announce(signedTransaction).subscribe(
    x => console.log(x),
    err => console.log(err)
);

$> nem2-cli transaction transfer --recipient SD5DT3-CH4BLA-BL5HIM-EKP2TA-PUKF4N-Y3L5HR-IR54 --mosaics foo:token::10 --message my_first_transfer_transaction
