########
Feedback
########

Following these guidelines helps to communicate that you respect the time of the people managing and developing this documentation. In return, they should reciprocate that respect in addressing your issue, assessing changes, and helping you finalise your pull requests.

***************
Error reporting
***************

Did you find a typo? Is something not explained clear enough?

Then, please open a Github |issue|. It is recommendable to review current opened issues, to see if someone already reported it.

If the error is not related to documentation, but with a specific project code, please include an issue in the project repository.

********
Requests
********

Many of the documentation and guides that NEM has today have been added because our users saw the need.
Open an |issue| on our issues list on GitHub which describes the missing documentation or improvement you would like to see.

If the improvement is not related to documentation, but with a specific project code, please include an issue in the project repository.

For improving NEM, please read |NIP|.

*********
Questions
*********

Please, don't use the issue tracker for support questions. You can join and present your specific questions and problems in the general discussion |slack| group.
