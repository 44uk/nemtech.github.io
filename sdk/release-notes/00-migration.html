Preparing for NEM2-SDK 1.0.0¶

We are pleased to present NEM-2SDK 0.9.1. These release notes include NEM2-SDK new features and breaking changes over its predecessor NEM-Library.

Warning

The SDKs methods could change until it reaches the stable version 1.0.0. For now, Catapult is only available in a test net environment. Consider it before applying changes and uploading your software to production.

Install latest version¶

Select your programming language and install the latest version. Make sure to uninstall previous versions of NEM2-SDK or NEM Library.

What is new?¶

NEM2-SDK shares the same design/architecture between programming languages. That means fast language adaptation, cohesion and shared knowledge between NEM developers, most rapid SDK updates and fewer bugs.

NEM-SDK does not add features that are not 100% related to NIS2-API. Instead, it permits to be extended by the use of new standalone libraries.

The new SDK implements the new features available in Catapult.

Aggregate transactions¶

Aggegate Transaction contains multiple transactions that can be initiated by different accounts. Use an Aggregate transaction when all transactions need to be included in a block or none of them.

Locks fund transactions¶

Locks funds transaction is used as a deposit to announce announced aggregate bonded transactions.

Once the related aggregate bonded transaction is confirmed, funds locked become available again in the account that had signed the initial lock funds transaction.

Multi-level multisig accounts (MLMA)¶

NEM2-SDK adds support to multi-level multisig accounts.

An MLMA is a multisig that has a cosigner that is another multisig.

Secret lock and secret proof transactions¶

Secret lock transaction is used to send mosaics to a recipient once an account discovers an attached secret message, known as proof.

Funds are unlocked and transferred when an account announces a valid Secret proof transaction.

Secret lock and proof transactions enable atomic cross-chain trading, without the necessity of trusting a third party.

Breaking changes¶

NEM Library inspires NEM2-SDK architecture. However, several architectural improvements have been made.

If you were using NEM-Library for NIS 1, consider reviewing the following breaking changes when considering upgrading to NEM2-SDK.

General

Package name

Import nem2-sdk rather than nem-library.

import { ... } from 'nem2-sdk';

Removed network bootstrapping

When using NEM Library, the network was selected when initiating a project.

In this new version, bootstrapping the network is not needed anymore.

NEM2-SDK allows you create transactions for different networks in the same project by providing the network when creating transactions.
